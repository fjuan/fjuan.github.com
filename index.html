My latest project: building a global Customer Support at Klarna

I joined Klarna in February 2015 to build the customer support web interface. We were a team of three developers plus a designer. Our task was to develop an application to fetch and merge information from several microservices and then present available actions to the Customer Support advisor.

The stack

The tool is split into a Ruby on Rails REST API plus a Javascript frontend built with Ember. Before the end of 2015 we included in our stack an Elasticsearch cluster in order to speed up the search of customers information.

Both Ruby and Javascript codebases have a thorough test coverage at different levels (unit, functional, integration) and are deployed following the principles of Continuos Integration and Continuos Delivery. While our test suits requires around 6 minutes to run, we improved our deployment pipeline to be able to release a new version in less than a minute after the test suite is verified. In total, it takes 8 minutes since the pull request is merged until the new code is in production and this has enabled us to release an average of 25 times per week.

The numbers

  • Elasticsearch: 10M documents, aprox 10Gb of data (by March 2016).
  • Test suit: 99,7% coverage, 6 minutes to run.
  • Deployments: < 2 minutes to production. Average of 25 times per week.
  • Uptime: 100% during working hours.
